## Key Hash Selection

The key hash identifies a specific gas lane — the maximum gas price the Chainlink node will use to fulfill your request. Guidelines:

- **Low gwei key hash** (2–30 gwei): Lower cost, may be slower during congestion. Good for low-urgency applications.
- **Medium gwei key hash** (100–500 gwei): Balanced speed/cost for most applications.
- **High gwei key hash** (1000 gwei): Fastest fulfillment even under network stress. Use for time-sensitive applications.

Always get the actual `bytes32` key hash values from the official docs or the VRF UI — they are not derived from the gwei value and must be copied exactly.
